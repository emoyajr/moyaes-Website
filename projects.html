Real-Time E-Commerce Fraud Detection Pipeline

Status: In Progress | Technologies: Python, Kafka, Spark Streaming, AWS, PostgreSQL, Redis, Docker

Building a real-time fraud detection system to process e-commerce transactions and identify suspicious activity as it occurs. The pipeline uses Apache Kafka for data streaming, Spark Streaming for real-time processing, and a dual-storage architecture with PostgreSQL for transaction history and Redis for high-speed lookups.

Implementing fraud detection algorithms that analyze transaction velocity, geographic patterns, purchase amounts, and behavioral anomalies. The system combines rule-based detection with statistical analysis to flag high-risk transactions for review.

Current Status: Developing core pipeline architecture and fraud detection logic. Performance metrics and results will be documented upon project completion.

Database Performance Optimization Study

Status: Planned | Technologies: PostgreSQL, ClickHouse, Python, Docker, Jupyter Notebooks

Planning a comprehensive performance comparison of row-based (PostgreSQL) and columnar (ClickHouse) database architectures for analytics workloads. The study will use large-scale datasets to systematically evaluate query performance, storage efficiency, and optimization strategies.

Will test multiple optimization techniques including indexing strategies, partitioning schemes, compression algorithms, and query tuning approaches. The research will measure performance across different workload patterns and data scales to understand real-world behavior and trade-offs.

Current Status: Research design and methodology in development. Benchmarking framework and detailed results will be published upon completion.
